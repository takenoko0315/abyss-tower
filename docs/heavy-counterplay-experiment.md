# 大技への複数対処法・小規模実験

## 対象

対象敵は10Fボスの**鉄の処刑人**。`attack → attack → heavy`という単純な固定パターンで、固有ギミックを
持たない。復活・毒・石化・特殊軽減などとの依存がなく、3つの対処法だけを切り分けやすく、最終ボスより
到達サンプルも得やすいため選定した。他の敵の大技は変更していない。

## 実装した対処法

- 大技を実際に防御してダメージを受け流した場合、重複しない「反撃態勢」を得る。次の直接攻撃全体を
  1.3倍し、攻撃開始時に1回だけ消費する。防御禁止時、通常攻撃・連攻への防御、回避だけで済んだ場合は得ない。
- 大技予告中の1プレイヤー行動に含まれる通常攻撃・スキル・会心・連撃・追加ヒットの直接ダメージ合計が
  敵最大HPの20%以上なら大技を中断する。毒・出血・炎上・棘・反動・過去ターンの継続ダメージは含めない。
- 大技予告中に気絶または凍結の残りターンが行動前より増えた場合、新規付与または延長成功として中断する。
  既存CCが残るだけ、同じターン数への再付与、付与失敗は中断しない。

判定は`src/game/heavyCounterplay.js`の純粋関数へ分離し、React側は結果をstate・ログ・SFXへ変換する。
予告欄には3対処法を短く表示する。

## Bot方針

- attack-only / greedy / basicは従来方針を維持。
- strategicは対象敵の大技に限り、確定付与型CC → 表示可能な攻撃力とスキル倍率から20%以上を見込める
  高火力行動 → 防御 → 合法行動、の順で選ぶ。確率CC、将来乱数、非表示の内部情報は使わない。

割合の分母は対象敵の大技予告回数。反撃態勢獲得は「防御ボタンを押した回数」ではなく、実際に大技を

## 判断

- 全体平均への影響は-0.04〜+0.08Fで小さく、他敵の挙動を変えていないことと整合する。
- greedyは予告を読まないまま偶発的に2回火力中断した。basicは仕様どおり防御中心、strategicは火力中断を
  2回選んだが83.3%は防御だった。
- この条件では**防御が事実上の固定最適解に近い**。戦士固定では確定CCスキルを持つ対象到達ランがなく、
  CC中断の比較材料を得られなかった。3手段の価値が拮抗したとは判断できない。
- 現時点では他敵へ展開しない。20%閾値や+30%を直ちに再調整する根拠も不足している。実プレイまたは別途合意した
  クラス/スキル固定実験でCCと火力の十分なサンプルを得てから、1回の再設計として検討する。

## 追加検証: 固定シナリオ・マイクロ計測

通常ランを追加せず、`scripts/heavy-counterplay-scenario.mjs`で鉄の処刑人戦だけを再現するNode専用モデルを
追加した。本番entryから参照されず、検証スクリプトや専用APIはproduction bundleへ入らない。既存の純粋関数
`calculateAttackDamage`、`calculateBaseIncomingDamage`、`rollAdditionalHits`、`resolveHeavyCounterplay`、
反撃態勢処理、実baselineと同じ`strategicPolicy`を直接使う。React・DOM・検証専用window APIは追加していない。

一方、これは実ゲームのReact戦闘を直接起動するE2Eではない。敵の通常/大技ターン、パターン進行、CD減少、
CCによる行動不能、勝敗判定はスクリプト内の簡易モデルである。装備アフィックス、クラス固有ゲージ、回復薬判断、
全状態異常、敵固有処理は再現しない。したがって「3分岐が成立するか」「20%判定・反撃態勢・Bot選択が
固定条件で整合するか」を調べるマイクロテストであり、実ゲームの勝率や全体バランスの推定値ではない。

実行例:

```text
npm run heavy-scenario -- --runs=100 --seed=7001 --workers=4
```

プレイヤーのクラス、HP/最大HP、攻撃/防御、スキル/CD、CC付与率、装備・フックと、敵HP/最大HP、

### strategicの選択

- A 防御型: 防御400/400、100%。火力・CCは選ばなかった。
- B 瞬間火力型: 火力中断200/200、100%。大技実発動0で、防御固定の勝率0%に対し100%、終了HP52。
- C CC型: CC選択300回のうち中断283回・同じ行動で撃破17回。CC利用可能300/300、大技実発動0で、
  防御固定の勝率0%に対し100%。撃破は中断成功へ重複計上しない。
- D 複合型: 全大技判断167回の内訳はCC 101回(60.5%)、火力33回(19.8%)、防御33回(19.8%)。
  初回判断は`cc-ready`が34/34 CC、`burst-ready`が33/33火力、`defense-only`が33/33防御だった。
  状況に応じた切り替えを再現できた。

### 追加確認の判断

- **防御は固定最適解ではない。** Aでは唯一の成立手段だが、B/Cでは防御固定が0勝、火力/CCが100勝。
  通常ランで防御が多かった主因は、対象到達時に火力・CC条件を満たすサンプルが少なかったことと判断する。
- **20%は到達不能ではない。** BとDの火力選択はすべて成功。予測誤差はBで平均+0.5・中央値+1ダメージ、
  D strategicで平均+0.55・中央値+1ダメージで、実ダメージを大きく過小・過大評価していない。
- **確定連撃を1行動として扱う。** 連撃率100%・単発22ダメージの境界テストで2ヒット合計による中断を確認。
  strategicの予測も画面表示される100%単位の確定追加ヒットだけを加味し、確率部分は見込まない。
- **CC失敗は中断にしない。** CC耐性100%のテストで、試行あり・CC中断0を確認。Cでは283中断+17撃破、
  D strategicでは34中断+67撃破で、選択した全ケースで大技発動を阻止した。CC利用可能性はCで100%、
  D strategicで101/167(60.5%)だった。
- `workers`は実プロセス数ではなく、seed列を分割する決定的チャンク数である。1と4でsummaryと全1600戦の
  生結果が完全一致し、分割方法に依存しないことを確認した。localStorage・グローバル乱数差し替えは使用しない。

### 結論

固定シナリオでは防御・火力・CCの各分岐がそれぞれ成立し、複合型strategicも条件別に選択を変えた。
ただしCの終了HP中央値4は余裕が非常に少なく、Dの勝率67%も機構成立を示すだけでバランス完成を意味しない。
現行の20%閾値と反撃態勢+30%に直ちに変更が必要な証拠はなく、数値は維持する。PR #6は鉄の処刑人限定の
実験としてマージ候補にできるが、他敵への展開や全体バランス判断は実ゲーム側の追加観測を別途必要とする。
