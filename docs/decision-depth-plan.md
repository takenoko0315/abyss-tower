# 判断の深さ設計ノート

[戦闘判断価値ベースライン](combat-decision-baseline.md)の計測結果をもとに、「毎ターンの判断」ではなく
「重要局面の判断」を濃くする方向でのゲームデザイン検討をまとめる。**このドキュメントは設計メモであり、
ここに書かれた実験案A〜Cはまだ実装していない。** ゲーム本体の数値・仕様・UIを変更する場合は、
別途このドキュメントを起点にした実装PRを立てて検討する。

## 現状診断

[combat-decision-baseline.md](combat-decision-baseline.md)(戦士固定・契約なし・300ラン、4方針比較)から:

- **攻撃とスキルを機械的に使うだけ(greedy)でも9.12Fまで進む**。大技/連攻を読む・防御するといった
  「深い」判断を一切せず、HPが減ったら回復薬、スキルが使えたら手近なものを使うだけで、
  何も考えないattack-only(6.98F)から大きく伸びる。**「まず殴ってスキルを回す」だけで到達階の
  大部分が説明できてしまう**、というのが現状の構造。
- **単純な「大技/連攻なら防御する」という1条件判断(basic)は平均を押し上げるが、個々のランでは
  ほぼ互角**。greedy→basicのペア比較は平均差+0.93F・95%信頼区間[0.36, 1.50]Fが0を跨がない
  一方、上回った割合48.3%・中央値差0.0F(下回った割合35.7%の方がむしろ僅かに多い)。「防御を
  覚えたら勝てるようになる」という体感には直結しにくい、薄い差。
- **HP・敵HP・状態異常・ギミック・スキル種別を組み合わせた複数判断(strategic)は比較的安定して有利**。
  basic→strategicは上回った割合54.0%・中央値+1.5Fと、平均だけでなく個々のランでも優位が出やすい。
- 上記3点を合わせると、**「毎ターンの判断」の価値は薄く、「重要局面でどう複数の情報を組み合わせるか」の
  判断の方が結果に効いている**、という構造が見えてくる。basicの「大技なら防御」は判断としては1個しか
  増えていないのに対し、strategicは同じ大技/連攻の場面でも「CCが使えるか」「敵は既に凍結/気絶して
  いないか」「毒牙持ちか」「HPはどれだけ余裕があるか」を見て行動を変えている。**判断の"回数"ではなく
  "重要な分岐点での選択肢の多さ"が差を生んでいる**、というのが今回のデータから読み取れる仮説。

なお、`combat-decision-baseline.md`の12節(RNGストリームの妥当性)で確認した通り、方針が分岐した
瞬間から同一シードでも展開が別物になっていくため、上記の差は現在のseed生成と方針分岐を含む
計測モデル上の比較であり、厳密な因果効果ではない。個々の対局を見比べての一問一答的な検証はできない前提で読むこと。

## 設計目標

上記の診断を踏まえ、次のバランス・敵行動設計(本ドキュメントの外で実施)が目指すべき方向性:

1. **通常ターンは攻撃でテンポを維持する**。雑魚戦・ボスの安全なターンまで毎回選択を強いると、
   greedyが既に取れている「攻撃とスキルを回すだけ」の心地よさを壊してしまう。
2. **重要ターンだけ複数の対処法を提示する**。strategicが優位に立てているのは「大技/連攻」のような
   限られた局面で複数の判断材料(CC・構え・防御・HP)を持っているからで、その局面の密度を上げる。
3. **同じ予告でもビルドによって正解が変わる**。CC持ちは妨害、構え系スキル持ちはカウンター、
   どちらもなければ防御、というふうに「唯一の正解」ではなく「持ち物に応じた正解」を用意する
   (strategicの優先順位2.が実際にこの形になっている)。
4. **正しい判断の利益を画面演出で明示する**。防御の軽減率・CCで足止めできたことなど、既に内部的に
   起きている効果を、プレイヤーが「今の判断が効いた」と実感できる形で見せる。
5. **攻撃以外の行動が次の火力につながるようにする**。防御→反撃・カウンターのように、守りの選択自体が
   次のターンの攻めに直結すると、「守るだけの薄い判断」から「守って攻める複合的な判断」に格上げできる。

## 小規模実験案

いずれも既存の大技/連攻・防御・ギミックの枠組みに乗せる想定の**小規模な調整案**。実装するかどうか・
どう数値化するかは別PRで検討する。

### 案A: 大技を一定ダメージまたはCCで中断可能にする

**狙い**: 現在の凍結/気絶は大技の予告そのものを打ち消さず、1ターン先送りにするだけ
(`combat-decision-baseline.md`の4節に記載)。「妨害すれば大技を消せる」という分かりやすい
因果関係を作ることで、CC所持の有無が明確な正解の分岐点になる。

- **プレイヤーの選択肢**: 大技の予告を見て「妨害する(CC/一定ダメージ)」か「受け止める準備をする
  (防御)」かを選ぶ。妨害に成功すれば予告そのものが消え、次のターンは通常状態に戻る。
- **ビルド別の正解**: CC持ち(frostnova/thunder)は妨害を優先。CCなしで火力に振ったビルドは
  「一定ダメージで中断」の閾値を狙って畳み掛けるか、素直に防御するかを選ぶ。防御特化ビルド
  (鉄壁の構え等)は妨害より受け止めの方が安定する場面が残る、というふうに複数の正解を維持したい。
- **悪用や固定最適解の危険**: 中断に必要なダメージ閾値が低すぎると「大技を見たら必ず殴って消す」が
  唯一の正解になり、かえって判断が単純化してしまう。閾値/CC成功率のチューニングを誤ると、防御・回避系の
  ビルド(硝子の魂などノーガードで攻め立てる契約)が一方的に有利/不利になる可能性もある。
- **必要な計測指標**: 「大技が中断された回数/大技が予告された回数」「中断成功時と通常防御時の被ダメ差」
  「CC所持ビルドと非所持ビルドの到達階差」。現行の`ccThreatTurnsSeen`(予告中のCC残存ターン数)とは
  別に「中断成功」を実測する指標を追加する形で計測基盤側は対応できる見込み。
- **最小実装範囲**: `enemyTurn`内で大技解決前に「予告後に一定ダメージを受けた/CCが新規に入った」かを
  判定し、条件を満たせば`intent`をリロールする分岐を1箇所追加する程度。既存の凍結/気絶ロジック
  (先送り)とは別の新しい効果として実装する必要がある。

### 案B: 大技を防御すると反撃ゲージまたは次攻撃ボーナスを得る

**狙い**: 現状も「防御反撃+15%」(`player.defendedLast`)は存在するが、大技/連攻に限定した
上乗せではなく防御全般の効果。大技という「重要ターン」に限定した特別な見返りを作ることで、
basicが到達できている「防御する」という浅い判断を、strategicのような「守って攻める」複合判断に
近づける。

- **プレイヤーの選択肢**: 大技/連攻を防御で受けるか、CC/構えで別の対処をするかに加え、「防御後の
  ボーナスをどう使うか」(次ターンに強い攻撃スキルを合わせる、など)という後続の選択が生まれる。
- **ビルド別の正解**: 火力スキルを持つビルドは「防御→ボーナス乗せでスキル」が強い選択になり、
  スキルが手薄なビルド(無音の誓い契約など)は通常攻撃にボーナスを乗せるだけでも十分な価値になる
  ようにボーナスの形(倍率か固定値か)を調整する必要がある。
- **悪用や固定最適解の危険**: ボーナスが大きすぎると「大技/連攻は必ず防御して踏み台にする」が
  唯一の正解になり、CC・構え系スキルの存在意義(2.3節参照)が薄れる。既存の「防御反撃+15%」との
  重複・スタックのさせ方も要検討(単純加算だと既存ビルドとの組み合わせで暴れる可能性)。
- **必要な計測指標**: 「大技防御後の次ターン与ダメージ」「大技防御を選んだ割合の変化(basic/strategic
  双方で防御への依存度が上がりすぎていないか)」「防御回数当たりの平均到達階への寄与」。
  既存の`mitigatedDamageEstimate`に加えて、防御後1ターン以内の与ダメ集計が新たに必要。
- **最小実装範囲**: `player.defendedLast`の倍率(+15%)を、直前の敵行動が`heavy`/`flurry`だった
  場合だけ引き上げる分岐を1箇所追加する程度。既存の防御反撃ロジックの延長として実装できる。

### 案C: 敵の防御中に集中・状態異常・準備行動の価値を作る

**狙い**: 現在、敵が構え中(guardTurns>0)の間は通常攻撃もスキルも一律で被ダメ半減されるだけで、
「構え中に何をすべきか」の選択肢が薄い(strategicも単に倍率の高いスキルを撃つだけ)。構え中を
「次の一撃に向けた準備ターン」として設計し直すことで、重要局面をもう1つ増やす。

- **プレイヤーの選択肢**: 構え中に「割引されると分かっていても攻撃で押す」か「状態異常を仕込む
  (被ダメ半減の影響を受けない毒/出血などのDoTを使う)」か「バフ/準備行動で次の構え解除後の一撃を
  強化する」かを選べるようにする。
- **ビルド別の正解**: 毒/出血スキル持ちは構え中にDoTを仕込むのが得、DoTを持たない純火力ビルドは
  構え解除を待って畳み掛けるのが得、というふうに手持ちスキルの種類で正解が分かれる形にしたい。
- **悪用や固定最適解の危険**: DoTが被ダメ半減を完全に無視できると「構え中は必ず毒/出血を撃つ」が
  固定解になり、他の選択肢が死ぬ。「半減の影響を受けにくいが完全に無視はしない」程度の軽い優遇に
  留める必要がある。また構え自体の意味(被ダメ半減で凌ぐターン)が薄れすぎないよう、敵側のバランスも
  同時に見る必要がある。
- **必要な計測指標**: 「構え中に選択された行動の内訳(攻撃/DoTスキル/準備行動)」「構え中DoT使用の
  有無による、その後の戦闘ターン数・被ダメの差」。既存の`guardTurnsSeen`/`attackVsGuardRate`を
  行動種別ごとに細分化する形で計測基盤を拡張できる。
- **最小実装範囲**: `targetGuarding`の適用対象からDoT付与系スキル(`applyStatus`を持つスキル)を
  除外する、または軽減率を緩和する分岐を`calculateAttackDamage`(`src/game/combat.js`)周辺に
  追加する程度。

## まとめ

3案とも、既存の大技/連攻・防御・ギミックという枠組みの中で「重要局面の選択肢を増やす」ことに絞った
小規模な調整であり、新しい戦闘システムを作るものではない。どれを実装するにしても、
`scripts/combat-decision-bot.mjs`の4方針(attack-only/greedy/basic/strategic)比較とペア比較の
仕組みがそのまま効果測定に使える設計にしてある(各案の「必要な計測指標」は既存の指標の延長で
収集できるものを優先して挙げた)。次のステップとしては、上記3案のうち1つを選んで小規模に実装し、
本ベースラインと同じ手法(固定シード・4方針・同一シードペア比較)で before/after を比較することを
想定している。
